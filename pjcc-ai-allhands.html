Selection + PF date as reference

🎯 Bet: unbiased choice reveals true cash-flow preference.

03 — The Process
Spec-Driven Development:
AI as the builder, humans as reviewers

What is SDD?

An approach where specifications — not code — are the primary artifact. Instead of "vibe coding" (throwing random prompts), you introduce deliberate intermediate steps, reviewed by humans, before the agent executes.

Why does it work?

By compressing context → cleaning → using only the summary, you keep the model's window focused for high-quality execution, without noise. Result: less technical debt, fewer AI "forgettings".

AI Hybrid vs AI Assist

Assist: AI as a task helper — humans still do everything.
Hybrid: AI executes autonomous flows — humans only review the output. Here, we reached Hybrid.

04 — Our Experience
What happened in practice
and what we learned
⚡
First specify round — incorrect result
AI used an unreviewed Confluence discovery and assumed information that was incorrect. Spec full of inaccuracies, many things missing.
Challenge: incomplete input → poor output
🎨
Kakau improved the Figma · Luana did the technical discovery
For the spec to work, the team needed to enrich the input artifacts: more detailed Figma with exact screens, and a new technical discovery document with cross-team dependencies.
Lesson: AI is only as good as the context it receives
🔄
Specify redone + corner case review
With improved artifacts, a new specify + clarify round identified edge cases: change screen before the success screen, PJ First customers always defaulting to day 1, rules when the due date fails.
Corner cases: human judgment still required
🚀
Plan → Tasks → Implement → PR → Staging
AI generated the plan, created tasks in Jira, implemented the code. Bruno reviewed directly in GitHub. Manual staging test, PR approval — and feature deployed.
2 days of development with near-zero human code

⚡ Challenges faced

Incomplete initial description led to the first incorrect spec — if the input isn't good, the output won't be either
Cross-team dependencies (billing, core) were undocumented — AI couldn't infer them
Business corner cases (e.g. PJ First, due date failure) require human judgment
Command sequence got confused — we deleted and re-ran the specify to clear the context

✅ What we learned

Quality specifications in the input are the most important investment in the process
2–3 days of spec prep → 2 days of dev → 1 week total
Real human work: specification, corner case review, staging test, PR approval
A 1-month project (2 sprints) done in 1 week with near-zero human implementation
